GIFPaint

Release 4

A simple HTML5 animation application inspired by MacPaint, Deluxe Paint and MSPaint.

Created by James Hicks, if you have any feedback or bugs you can get in touch on Twitter or info (at symbol) gifpaint.com. I'd also love to see any cool GIFs you make using GIFPaint and will reblog them to the GIFPaint Tumblog

GIFPaint was made possible using the following Open Source projects: Processing.js, omggif, jsgif, FileReader.js, jQuery

Using GIFPaint

GIFpaint is very much a work in progress, some things might break/change/improve in the coming weeks.

All tools in GIFPaint use both mouse buttons, Left Button draws/fills/grabs the first colour well and Right Button uses the second colour well

GIFPaint has shortcut keys for most things:

  • Left Arrow - Previous Frame
  • Right Arrow - Next Frame
  • Z - Undo
  • A - Add Frame
  • C - Copy Frame
  • P - Paste Frame
  • B - Brush Tool
  • F - Fill Bucket Tool
  • I - Ink Dropper Tool
  • T - Take Webcam Photo
  • + - Increase Brush Size
  • - - Decrease Brush Size
  • X - Swap left and right colours
  • [ and ] - Toggle Onion Skins

Shortcut keys in GIFPaint are simply the keys, you don't need to hold Command or CTRL. Hold shift and click with the Brush Tool to draw straight lines.

Saving Files

Multiple GIFs can be encoded in parallel, once encoded they are not saved anywhere so if you wish to keep them click Download or login to Tumblr and post them straight to Tumblr.

Opening Files

Drag any GIF to the canvas and your current document will be replaced by a blank document with the GIF on a tracing layer. The tracing layer can be toggled or changed in opacity

If the tracing layer is visible when you hit encode, then it will be encoded into the GIF. Keep in mind the current GIF encoder doesn't handle photos well, this will be addressed in the future.

Any JPG or PNG dragged to the canvas will be added to the current tracing layer. You can also use CMD+V on Macs and CTRL+V on Windows to paste the current contents of the clipboard if its an image.

Known Issues

GPU accelerated 2D canvas causes errors with the Fill Tool with some graphics cards in Chrome. Type "chrome://flags/" into the address bar and check "Disable accelerated 2D canvas" to fix this and speed up drawing in general.

Webcam captures look better with smaller canvas sizes.

Webcam currently only works in Chrome
